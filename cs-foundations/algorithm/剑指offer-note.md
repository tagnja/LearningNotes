<h2><a name="intv_proc_t" href="#intv_proc_c">1. 面试流程</a></h2>
<br>

### 1.1 面试官谈面试

- 初级程序员偏向考察算法和数据结构，看应聘者的基本功；对于高级程序员，我会多关注专业技能和项目经验。
- 应聘者要事先做好准备，对公司近况、项目情况有所了解，对应聘的工作很有热情。还要准备好合适的问题问面试官。
- 面试过程中，首先需要放松，其次不要基于编写代码，写完代码不要马上提交，自己检查和测试一下。

### 1.2 面试的3种形式

电话面试

共享桌面远程面试

- 想清楚再开始编码
- 良好的代码命名和缩进对齐习惯。
- 能够进行单元测试。

现场面试，应做好以下准备

- 规划好路线和估算出行时间。
- 准备好得体的衣服。
- 注意面试邀请函里的面试流程。
- 准备几个问题。应聘者提问的问题。

### 1.3 面试的3个环节

#### 行为面试环节

**自我介绍**

应聘者的项目经验

- Situation：简短的项目背景。项目的规模，开发软件的功能，目标用户等。
-  Task：自己完成的任务。参与xxx，负责xxx。
- Action：为完成任务做了哪些工作，怎么做的。详细介绍。基于什么平台，应用了什么技术等。
- Result：自己的贡献。最好能用数字加以说明。完成了多少功能；优化，性能提高了多少百分比；维护，修复了多少个 Bug。

针对项目经验最常问的问题：

- 你在该项目中碰到的最大问题是什么，你是怎么解决的？
- 从这个项目中你学到了什么?
- 什么时候会和其他团队成员（包括开发、测试、设计、项目经理等）有什么样的冲突，你们是怎么解决冲突的？

**应聘者掌握的技能**

- 了解。某项技术上过课或者看过书，没有做过实际的项目。不建议写在简历中，除非对应聘的岗位需要。
- 熟悉。简历中大部分应该是熟悉。在实际项目中使用某项技术很长时间，通过查阅文档可以独立解决大部分问题。毕业设计用到的技术，工作项目开发中用到的技术，可以用熟悉。
- 精通。能够轻松回答这个领域的绝大多数问题。不要轻易在简历上写精通。

**回答为什么跳槽**

避免以下4个原因

- 老板太苛刻。
- 同事太难相处。
- 加班太频繁。
- 工资太低。

正确回答的参考：

- 对现在的工作有些厌倦，没有激情。
- 对某些方面没有太多兴趣。
- 主要工作是维护和修复 Bug，软件开发和设计能力得不到提升。

#### 技术面试环节

5个方面的素质：

- 扎实的基础知识。包括编程语言，数据结构和算法等。
- 高质量的代码。能写出正确的、完整的、鲁棒的高质量代码。
- 清晰的思路。能思路清晰地分析、解决问题。
- 优化效率的能力。能从时间、空间复杂度两方面优化算法效率。
- 优秀的综合能力。具备优秀地沟通能力、学习能力、发散思维能力等。

#### 应聘者提问

不适合问的问题

- 不要问和自己的职位没有关系的问题。如公司未来五年的发展战略是什么。
- 不要问薪水。工资一般是通过面试后跟 HR 谈。
- 不要立即打听面试结果。

推荐问的问题

- 与应聘的职位或项目相关的问题。如，主要做什么工作。需要学习相关的知识吗。
- 了解公司相关信息再提问。如公司成立时间，主要的业务，职位要求等。
- 留心面试官说过的话。

<h2><a name="intv_basic_knowle_t" href="#intv_basic_knowle_c">2. 面试需要的基础知识</a></h2>
<br>

### 2.1 面试官谈基础知识

- C++ 的基础知识，如面向对象、构造函数、析构函数、动态绑定等，能够反映应聘者是否善于把握本质，有没有耐心深入一个问题。另外还有常用的设计模式、UML图等，体现应聘者是否有软件工程的经验。
- 对于基础知识的考察，我特别重视 C++ 中对内存的使用管理。
- 基础知识反映了一个人的基本能力和基础素质。我一般考察：（1）数据结构和算法；（2）编程能力；（3）部分数学知识，如概率；（4）问题分析和推理能力。
- 我比较重视四块基础知识：（1）编程基本功，如字符串处理问题；（2）并发控制；（3）算法、复杂度；（4）语言的基本概念。
- 我会考察编程基础、计算机系统基础知识、算法及设计能力。这些时软件工程师最基本的要求。
- （1）对 OS 的理解度。对于 OS 理解比较深入的人对于偏底层的工作上手一般比较快。（2）对于一门编程语言的掌握程度。对某种语言比较深入了解，对新的编程语言上手也比较快。（3）常用的算法和数据结构。不了解这些只能写写 Hello World。

综合来说

- 数学能力。
- 算法和编程能力。
- 编程语言语法特性的深入理解。
- 计算机基础的深入理解。
- 软件工程能力。
- 其他综合能力。

### 2.2 编程语言

面试中会至少设计一种编程语言。

面试过程中，考察编程语言：要么直接问语言的语法，要么让应聘者用一种编程语言写代码解决一个问题。

流行的编程语言有很多，不同公司开发用的语言也不尽相同。常见的语言和其应用场景如：

- 底层开发：C。
- Linux 下开发：C++。
- 基于 Windows：C#。
- 跨平台开发：Java。
- 基于苹果系统开发：Objective C。
- 短小精致的小应用软件：脚本语言 Perl、Python。

### 2.3 数据结构

数据结构一直是技术面试的重点，面试大多数围绕数组、字符串、链表、树、栈及队列这几种常见的数据结构。每一个都需要熟练掌握。

数组和字符串是两种最基本的数据结构，它们是连续内存。链表和树是面试种出现频率最高的数据结构。由于操作链表和树需要大量的指针，一定要留意代码的鲁棒性。栈和队列可以帮助我们高效的解决很多问题。

数组

- 由于数组种的内存是连续的，可以根据下标 O(1) 时间读写任何元素，可以使用数组来是实现简单的哈希表。

链表

- 链表的结构很简单，代码量比较适合面试。
- 链表是一种动态的数据结构，需要比较好的编程功底才能写出完整的操作链表的代码。
- 链表比较灵活，可以用链表来设计具有挑战性的面试题。

树

- 树的操作会涉及大量的指针，面试中往往利用树来考察在复杂指针操作情况下写代码的能力。面试中大部分是二叉树。
- 二叉树有3种遍历方式：前序、中序和后序。每一种哦遍历都有递归和循环两种不同的实现方式。另一种，按层遍历比较少用。
- 二叉树有很多特例。如二叉搜索树，堆，和红黑树。

### 2.4 算法和数据操作

- 很多算法都可以用递归和循环两种不同的实现方式。通常基于递归的实现方式代码会比较简洁，但性能不如基于循环的实现方式。
- 通常排序和查找是面试时的考察算法的重点。重点掌握二分查找、归并排序和快速排序，做到能随时正确、完整地写出它们地代码。
- 如果要求在二维数组（表现为迷宫或者棋盘等）上搜索路径，我们可以尝试使用回溯法。回溯法很适合用递归的代码实现，我们也可以用栈来模拟递归过程。
- 如果要求某个问题的最优解，并且该问题可以分为多个子问题，那么我们可以尝试用动态规划。
- 如果在分解子问题的时候存在某个特殊的选择，采用这个选择一定能得到最优解，那么，可以尝试使用贪婪算法。
- 位运算可以看成一类特殊的算法。位运算有与、或、异或、左移和右移5种位运算。

循环和递归

- 递归的代码比较简洁，但性能没有循环好。如果面试官没有特殊的要求，应聘者可以优先采用递归的方式编程。
- 递归的缺点表现为：每一次函数调用，都需要在内存栈中分配空间以保存参数、返回地址和临时变量，而且往栈里压入数据和弹出数据都需要时间。递归还有可能导致调用栈溢出。

查找和排序

- 查找相对比较简单，不外乎顺序查找、二分查找、哈希表查找和二叉树查找。
- 排序比查找要复杂一些。面试总经常要求比较插入排序、冒泡排序、归并排序、快速排序等不同算法的优劣。
- 面试时，一定要对各种排序算法的特点非常熟悉，能够从额外空间消耗、平均时间复杂度和最差时间复杂度等方面比较它们的优缺点。
- 面试经常要求写出快速排序的代码。

<h2><a name="high_qual_code_t" href="#high_qual_code_c">3. 高质量代码</a></h2>
<br>

### 3.1 面试官谈高质量的代码

- 一般考察应聘者对代码的容错处理能力。不能容忍只针对假想的正常值进行处理，不考虑异常状况，也不考虑资源回收等问题。
- 如果是粗心犯错，则可以原谅，因为毕竟面试的时候会紧张；不能容忍的是，该掌握的知识点却没有掌握，而且提醒了还不知道。如 double d1,d2; if(d1 == d2)
- 最不能容忍功能错误，忽略边界情况。
- 如果一个程序员连变量、函数命名都毫无章法，解决一个具体的问题都找不到一个最合适的数据结构，那么这回让面试官对他的印象大打折扣。这只能说明他程序写得太少，不够熟悉。
- 我会从程序的正确性和鲁棒性两个方面检验代码的质量。会关注对输入参数的检查、处理错误和异常的方式、命名方式等。

### 3.2 代码的规范性

- 清晰的书写。手写代码时，减慢速度，写清楚时很有必要的。
- 清晰的布局。合理的缩进，括号对齐。
- 合理的命名。用完整的英文单词组合命名变量和函数。

### 3.3 代码的完整性

- 完成了基本的功能。
- 输入边界值能够得到正确的输出。
- 对不合规范的非法输入做出合理的错误处理。

保证代码完整性的测试

- 功能测试。
- 边界测试。输入的边界值是否得到正常输出。代码中的循环的边界是否正确。递归终止的边界值是否正确。字符串转换为数字和数字算术运算是否溢出数值边界。
- 负面测试。考虑错误输入。

错误处理的方法

- 函数用返回值告知调用者是否出错。很多 API 返回值为0表示成功，不为0表示错误。
- 设置一个全局变量。
- 抛出异常。有些编程语言不支持抛出异常。

### 3.4 代码的鲁棒性

鲁棒性（Robust）是指程序能够判断输入是否合乎规范要求，并对不符合要求的输入予以合理的处理。它属于完整性中的负面测试。

总结

高质量代码主要考虑规范性和完整性（功能、边界、负面）。
